Шпаргалка для Git

Basic

git config --global user.name blabla - команда при реєстрації в gid додає імя користувача

git config --global user.email blabal@bla команда при реєстрації в gid додає електронну пошту користувача

git init- створює локальний репозиторій

git status-Команда git status показывает состояния файлов в рабочем каталоге и индексе: какие файлы изменены, но не добавлены в индекс; какие ожидают коммита в индексе. Вдобавок к этому выводятся подсказки о том, как изменить состояние файлов.

git add-Команда git add добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита.

git commit-Команда git commit берёт все данные, добавленные в индекс с помощью git add, и сохраняет их слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок.

git log- показує образно історію репозиторія що з ним відбувалось

git checkout- відміняє зміну файла

git branch- команда дозволяє створювати, переглядати, перейменувати, удаляти гілки репозиторія

git push-Команда git push чаще всего используется для публикации выгружаемых локальных изменений в центральном репозитории

git pull- команда зазвичай дістає дані з серверу, з якого ви зробили клон, та намагається злити її з кодом, над яким ви зараз працюєте

git merge- используется для слияния одной или нескольких веток в текущую.

git clone- это утилита командной строки Git для выбора существующего репозитория и создания его клона

.gitignore- Файл із розширенням файлу GITIGNORE - це файл Git Ignore, який використовується у системі управління версією / джерелом під назвою Git. Він визначає, git файли та папки не слід ігнорувати у заданому вихідному коді.

Advanced

git revert - скасовує деякі існуюючі коміти

git reset - використовується для скасування змін до заданого стану

git rebase - один з двох способів об'єднання змін, зроблених в одній гілці зі змінами в іншій гілці

git cherry-pick- застосовує зміни, внесені деякими існуючими комітами

git stash - бере чорновий стан вашої робочої директорії — тобто ваші змінені супроводжувані файли та індексовані зміни — та зберігає їх у стеку незавершених змін, які ви можете знову використати будь-коли (і навіть на іншій гілці).
